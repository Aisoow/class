## 1.2 題 - 2補數與1補數

**題目：** 試分別求出下列各二進制數目的2補數與1補數

### (1) 01101101

**✅ 答案：**
- **1補數：** 10010010
- **2補數：** 10010011

**📝 解題步驟：**

**1補數計算：** 所有位元反轉（0↔1）
```
原數：0 1 1 0 1 1 0 1
反轉：1 0 0 1 0 0 1 0
```

**2補數計算：** 1補數 + 1
```
  10010010  (1補數)
+        1
----------
  10010011  (2補數)
```

### (2) 101011.0101

**✅ 答案：**
- **1補數：** 010100.1010
- **2補數：** 010100.1011

**📝 解題步驟：**

**1補數計算：** 所有位元反轉
```
原數：1 0 1 0 1 1 . 0 1 0 1
反轉：0 1 0 1 0 0 . 1 0 1 0
```

**2補數計算：** 1補數 + 0.0001
```
  010100.1010  (1補數)
+ 000000.0001
-------------
  010100.1011  (2補數)
```

**💡 重點提醒：**
- 小數點位置固定不變
- 加1從最低位（最右邊）開始

### (3) 101101101

**✅ 答案：**
- **1補數：** 010010010
- **2補數：** 010010011

**📝 解題步驟：**

**1補數計算：** 所有位元反轉（0↔1）
```
原數：1 0 1 1 0 1 1 0 1
反轉：0 1 0 0 1 0 0 1 0
```

**2補數計算：** 1補數 + 1
```
  010010010  (1補數)
+         1
-----------
  010010011  (2補數)
```

### (4) 0.10101

**✅ 答案：**
- **1補數：** 1.01010
- **2補數：** 1.01011

**📝 解題步驟：**

**1補數計算：** 所有位元反轉
```
原數：0 . 1 0 1 0 1
反轉：1 . 0 1 0 1 0
```

**2補數計算：** 1補數 + 0.00001
```
  1.01010  (1補數)
+ 0.00001
---------
  1.01011  (2補數)
```

**💡 重點提醒：**
- 純小數的2補數計算時，從最右邊（最低位）加1
- 小數點位置固定不變

---

**🔑 相關概念整理：**
- **1補數 (One's Complement)：** 將二進制數的每個位元反轉
- **2補數 (Two's Complement)：** 1補數加1，是現代電腦最常用的有號數表示法
- **2補數優點：** 只有一個零的表示法，加減運算統一處理

## 1.5 題 - 十進制轉二進制

**題目：** 轉換下列各十進制數目為二進制

### (1) 17

**✅ 答案：** 10001₂

**📝 解題步驟（除2取餘法）：**
```
17 ÷ 2 = 8 ... 餘 1  ↑
 8 ÷ 2 = 4 ... 餘 0  |
 4 ÷ 2 = 2 ... 餘 0  | 由下往上讀
 2 ÷ 2 = 1 ... 餘 0  |
 1 ÷ 2 = 0 ... 餘 1  ↓
```
**結果：** 10001₂

**✓ 驗證：** 1×16 + 0×8 + 0×4 + 0×2 + 1×1 = 17 ✓

### (2) 231

**✅ 答案：** 11100111₂

**📝 解題步驟：**
```
231 ÷ 2 = 115 ... 餘 1  ↑
115 ÷ 2 = 57  ... 餘 1  |
 57 ÷ 2 = 28  ... 餘 1  |
 28 ÷ 2 = 14  ... 餘 0  | 由下往上讀
 14 ÷ 2 = 7   ... 餘 0  |
  7 ÷ 2 = 3   ... 餘 1  |
  3 ÷ 2 = 1   ... 餘 1  |
  1 ÷ 2 = 0   ... 餘 1  ↓
```
**結果：** 11100111₂

**✓ 驗證：** 128 + 64 + 32 + 4 + 2 + 1 = 231 ✓

### (3) 780.75

**✅ 答案：** 1100001100.11₂

**📝 解題步驟：**

**整數部分 780（除2取餘法）：**
```
780 ÷ 2 = 390 ... 餘 0  ↑
390 ÷ 2 = 195 ... 餘 0  |
195 ÷ 2 = 97  ... 餘 1  |
 97 ÷ 2 = 48  ... 餘 1  |
 48 ÷ 2 = 24  ... 餘 0  | 由下往上讀
 24 ÷ 2 = 12  ... 餘 0  |
 12 ÷ 2 = 6   ... 餘 0  |
  6 ÷ 2 = 3   ... 餘 0  |
  3 ÷ 2 = 1   ... 餘 1  |
  1 ÷ 2 = 0   ... 餘 1  ↓
```
整數部分：**1100001100₂**

**小數部分 0.75（乘2取整法）：**
```
0.75 × 2 = 1.50 → 取整數 1 ↓
0.50 × 2 = 1.00 → 取整數 1 | 由上往下讀
```
小數部分：**.11₂**

**完整結果：** 1100001100.11₂

**✓ 驗證：** 
- 整數：512+256+64+8+4 = 780 ✓
- 小數：0.5+0.25 = 0.75 ✓

### (4) 1103.31

**✅ 答案：** 10001001111.010011111...₂（循環小數）

**📝 解題步驟：**

**整數部分 1103（除2取餘法）：**
```
1103 ÷ 2 = 551 ... 餘 1  ↑
 551 ÷ 2 = 275 ... 餘 1  |
 275 ÷ 2 = 137 ... 餘 1  |
 137 ÷ 2 = 68  ... 餘 1  |
  68 ÷ 2 = 34  ... 餘 0  | 由下往上讀
  34 ÷ 2 = 17  ... 餘 0  |
  17 ÷ 2 = 8   ... 餘 1  |
   8 ÷ 2 = 4   ... 餘 0  |
   4 ÷ 2 = 2   ... 餘 0  |
   2 ÷ 2 = 1   ... 餘 0  |
   1 ÷ 2 = 0   ... 餘 1  ↓
```
整數部分：**10001001111₂**

**小數部分 0.31（乘2取整法）：**
```
0.31 × 2 = 0.62 → 取 0 ↓
0.62 × 2 = 1.24 → 取 1 |
0.24 × 2 = 0.48 → 取 0 | 由上往下讀
0.48 × 2 = 0.96 → 取 0 |
0.96 × 2 = 1.92 → 取 1 |
0.92 × 2 = 1.84 → 取 1 |
0.84 × 2 = 1.68 → 取 1 |
0.68 × 2 = 1.36 → 取 1 |
0.36 × 2 = 0.72 → 取 0 ↓
...（持續循環，無法精確表示）
```
小數部分：**.010011111...₂**

**完整結果：** 10001001111.010011111...₂

**💡 重點提醒：** 0.31無法用有限位數的二進制精確表示，會產生循環小數

---

**🔑 相關概念整理：**
- **整數轉二進制：** 除2取餘法，餘數由下往上讀
- **小數轉二進制：** 乘2取整法，整數部分由上往下讀
- **循環小數：** 某些十進制小數（如0.1, 0.31）無法精確轉換為二進制

## 1.18 題 - 二進制減法

**題目：** 分別使用1補數與2補數表示法執行下列各正的二進制數目的減法運算

### (1) 11010 - 11001

**✅ 答案：** 00001₂ = 1₁₀

**📝 解題步驟：**

**方法一：2補數法**
```
步驟1：求11001的2補數
  11001 → 00110（反轉）→ 00111（+1）

步驟2：加法運算
  11010（被減數）
+ 00111（減數的2補數）
-------
 100001 → 00001（最高位進位丟棄）
```

**方法二：1補數法**
```
步驟1：求11001的1補數
  11001 → 00110（反轉）

步驟2：加法運算
  11010（被減數）
+ 00110（減數的1補數）
-------
 100000
+     1（循環進位，加回最低位）
-------
  00001
```

**✓ 驗證：** 26 - 25 = 1 ✓

### (2) 11010 - 10000

**使用2補數方法：**
```
被減數：11010
減數的2補數：10000（10000的2補數）

  11010
+ 10000
-------
 101010  （進位丟棄）
結果：01010₂ = 10₁₀
```

**使用1補數方法：**
```
被減數：11010
減數的1補數：01111

  11010
+ 01111
-------
 101001
+     1  （循環進位）
-------
  01010
結果：01010₂ = 10₁₀
```

**驗證：** 26 - 16 = 10 ✓

### (3) 1010.011 - 101.0111

**使用2補數方法：**
```
對齊小數點：
  1010.0110
-  101.0111
-----------
被減數：    1010.0110
減數的2補數：1010.1001

  1010.0110
+ 1010.1001
-----------
 10100.1111  （進位丟棄）
結果：0100.1111₂
```

**使用1補數方法：**
```
被減數：    1010.0110
減數的1補數：1010.1000

  1010.0110
+ 1010.1000
-----------
 10100.1110
+        1  （循環進位）
-----------
  0100.1111
結果：0100.1111₂
```

**解析：**
- 10.1875 - 5.4375 = 5.25 或更精確地說是約4.9375
- 小數點要對齊進行運算

### (4) 11101.01101 - 1011.01

**使用2補數方法：**
```
對齊：
  11101.01101
-  1011.01000
-------------
被減數：     11101.01101
減數的2補數：10100.11000

  11101.01101
+ 10100.11000
-------------
 110010.00101  （進位丟棄）
結果：10010.00101₂
```

**使用1補數方法：**
```
被減數：     11101.01101
減數的1補數：10100.10111

  11101.01101
+ 10100.10111
-------------
 110001.00100
+          1  （循環進位）
-------------
  10010.00101
結果：10010.00101₂
```

**驗證：** 29.40625 - 11.25 = 18.15625 ✓

**相關概念：**
- **2補數減法：** A - B = A + (B的2補數)，最高位進位直接丟棄
- **1補數減法：** A - B = A + (B的1補數)，循環進位（end-around carry）加回最低位
- 2補數法較簡單，無需處理循環進位

## 1.20 題 - 2補數加法(8位元)

**題目：** 使用2補數表示法計算下列各小題（假設8位元）

### (1) 37 + 45

**解答：** 82，無溢位

**解析：**
```
37₁₀ = 00100101₂
45₁₀ = 00101101₂

  00100101
+ 00101101
----------
  01010010 = 82₁₀
```

**結果分析：**
- 最高位為0，表示正數
- 無溢位發生（兩個正數相加得正數）
- 答案：82

### (2) 75 + 98

**解答：** 173，無溢位（在無號數系統中）

**解析：**
```
75₁₀  = 01001011₂
98₁₀  = 01100010₂

  01001011
+ 01100010
----------
  10101101 = -83₁₀（有號數）或 173₁₀（無號數）
```

**結果分析：**
- 在**有號數系統**：溢位！（兩正數相加得負數，不合理）
  - 最高位為1，表示負數
  - 實際結果超出8位元有號數範圍（-128~127）
- 在**無號數系統**：173（正確）
  - 8位元無號數範圍：0~255

**相關概念：溢位檢測**
- 兩個正數相加得負數 → 溢位
- 兩個負數相加得正數 → 溢位

### (3) 47 + 78

**解答：** 125，無溢位

**解析：**
```
47₁₀ = 00101111₂
78₁₀ = 01001110₂

  00101111
+ 01001110
----------
  01111101 = 125₁₀
```

**結果分析：**
- 最高位為0，表示正數
- 無溢位（在有號數範圍內）
- 答案：125

### (4) 75 + 23

**解答：** 98，無溢位

**解析：**
```
75₁₀ = 01001011₂
23₁₀ = 00010111₂

  01001011
+ 00010111
----------
  01100010 = 98₁₀
```

**結果分析：**
- 最高位為0，表示正數
- 無溢位
- 答案：98

**相關概念：**
- **8位元有號數範圍：** -128 ~ +127
- **8位元無號數範圍：** 0 ~ 255
- **溢位判斷：** 符號位（最高位）變化不合邏輯時發生

## 1.21 題 - 2補數有號數運算(8位元)

**題目：** 使用2補數表示法計算下列各小題（假設8位元）

### (1) 37 - (-45) = 37 + 45

**✅ 答案：** 82

**📝 解題步驟：**

減去負數等於加上正數，所以 37 - (-45) = 37 + 45

```
37₁₀ = 00100101₂
45₁₀ = 00101101₂

  00100101
+ 00101101
----------
  01010010 = 82₁₀
```

**✓ 驗證：** 37 + 45 = 82 ✓

### (2) 35 + (-36)

**✅ 答案：** -1

**📝 解題步驟：**

首先求-36的2補數表示：
```
36₁₀ = 00100100₂
反轉：  11011011
加1：  +       1
-36₂ = 11011100₂
```

執行加法運算：
```
35₁₀ = 00100011₂
-36₁₀ = 11011100₂

  00100011
+ 11011100
----------
  11111111 = -1₁₀
```

**✓ 驗證：** 11111111的2補數 = 00000001 = 1，所以11111111 = -1 ✓

### (3) (-37) - (-45) = -37 + 45

**✅ 答案：** 8

**📝 解題步驟：**

減去負數等於加上正數，所以 (-37) - (-45) = -37 + 45

首先求-37的2補數表示：
```
37₁₀ = 00100101₂
反轉：  11011010
加1：  +       1
-37₂ = 11011011₂
```

執行加法運算：
```
-37₁₀ = 11011011₂
45₁₀ = 00101101₂

  11011011
+ 00101101
----------
 1 00001000  （最高位進位丟棄）
= 00001000 = 8₁₀
```

**✓ 驗證：** -37 + 45 = 8 ✓

### (4) (+75) - (-45) = 75 + 45

**✅ 答案：** 120

**📝 解題步驟：**

減去負數等於加上正數，所以 75 - (-45) = 75 + 45

```
75₁₀ = 01001011₂
45₁₀ = 00101101₂

  01001011
+ 00101101
----------
  01111000 = 120₁₀
```

**✓ 驗證：** 75 + 45 = 120 ✓

**相關概念：**
- **減去負數 = 加正數：** A - (-B) = A + B
- **負數的2補數表示：** 正數取2補數得負數
- **最高位進位丟棄：** 在有號數運算中，超出位元範圍的進位直接丟棄

## 1.22 題 - BCD碼轉換

**題目：** 轉換下列各數為BCD碼

### (1) 10110111011₂

**解答：** 先轉十進制，再轉BCD

**解析：**
```
10110111011₂ = ?₁₀

1×2¹⁰ + 0×2⁹ + 1×2⁸ + 1×2⁷ + 0×2⁶ + 1×2⁵ + 1×2⁴ + 1×2³ + 0×2² + 1×2¹ + 1×2⁰
= 1024 + 256 + 128 + 32 + 16 + 8 + 2 + 1
```

<a id="重點概念總結"></a>
## 📚 重點概念總結

### 🔢 1. 補數系統
**1補數 (One's Complement)**
- 計算方式：所有位元反轉（0↔1）
- 特點：有+0和-0兩種表示

**2補數 (Two's Complement)**
- 計算方式：1補數 + 1
- 應用：現代電腦有號數表示的標準方法
- 優點：
  - 只有一個零的表示
  - 加減法運算統一處理
  - 無需特殊處理符號位

**減法運算**
- 2補數法：A - B = A + (B的2補數)，最高位進位丟棄
- 1補數法：A - B = A + (B的1補數)，循環進位加回最低位

---

### 🔄 2. 數制轉換

**十進制 → 二進制**
- **整數部分：** 除2取餘法，餘數由下往上讀
- **小數部分：** 乘2取整法，整數部分由上往下讀
- **注意：** 某些十進制小數（如0.1, 0.31）無法精確轉為二進制

**轉換公式**
| 進制 | 基數 | 使用符號 |
|------|------|----------|
| 二進制 | 2 | 0, 1 |
| 八進制 | 8 | 0-7 |
| 十進制 | 10 | 0-9 |
| 十六進制 | 16 | 0-9, A-F |

---

### 💾 3. BCD碼 (Binary-Coded Decimal)

**編碼規則**
- 每個十進制數字用4位元二進制表示
- 有效範圍：0000(0) ~ 1001(9)
- 無效碼：1010~1111

**優缺點**
- ✅ 優點：與十進制轉換簡單，適合顯示器
- ❌ 缺點：儲存效率低（浪費6個編碼）

**轉換步驟**
1. 將原數轉為十進制
2. 每個十進制數字轉4位元BCD

---

### 🔀 4. 格雷碼 (Gray Code)

**特性**
- 相鄰兩數只有一個位元不同
- 又稱循環二進制碼或反射二進制碼

**應用場景**
- 旋轉編碼器
- 類比轉數位轉換器（ADC）
- 避免多位元同時變化造成的錯誤讀數

**轉換公式**
- **Binary → Gray:** Gᵢ = Bᵢ ⊕ Bᵢ₊₁（最高位不變，其餘位XOR前一位）
- **Gray → Binary:** Bᵢ = Bᵢ₊₁ ⊕ Gᵢ（從高位累積XOR）

---

### 📝 5. ASCII碼

**編碼範圍**
| 類型 | ASCII值 | 十六進制 |
|------|---------|----------|
| 數字 '0'-'9' | 48-57 | 30-39 |
| 大寫 'A'-'Z' | 65-90 | 41-5A |
| 小寫 'a'-'z' | 97-122 | 61-7A |
| 空格 | 32 | 20 |

**重要特性**
- 標準ASCII：7位元（0-127）
- 大小寫字母相差32（第5位元不同）
- 大寫轉小寫：+32 或 OR 0x20
- 小寫轉大寫：-32 或 AND 0xDF

---

### ⚠️ 6. 溢位檢測

**8位元有號數範圍**
- 範圍：-128 ~ +127
- 最高位（MSB）為符號位：0=正，1=負

**溢位判斷規則**
| 運算 | 符號變化 | 是否溢位 |
|------|----------|----------|
| 正 + 正 | 結果為負 | ✓ 溢位 |
| 負 + 負 | 結果為正 | ✓ 溢位 |
| 正 + 負 | 任何結果 | ✗ 不溢位 |
| 相同符號相減 | 任何結果 | ✗ 不溢位 |

<a id="學習建議與考試技巧"></a>
## 💡 學習建議與考試技巧

### ✅ 必須熟練掌握
1. **補數計算：** 2補數是基礎，務必練到不用思考就能算出
2. **進制轉換：** 二、八、十、十六進制互轉要快速準確
3. **格雷碼轉換：** 記住XOR規則，從高位往低位依序計算
4. **ASCII常用值：** 至少要記住數字和字母的範圍

### 📋 解題檢查清單
- [ ] 補數運算後有沒有加1？
- [ ] 進制轉換的餘數/整數順序對了嗎？（整數由下往上，小數由上往下）
- [ ] 小數點對齊了嗎？
- [ ] 最高位進位處理正確嗎？（2補數丟棄，1補數循環）
- [ ] BCD每個數字都是4位元嗎？
- [ ] 格雷碼最高位記得保持不變嗎？

### 🎯 快速記憶口訣
- **1補數：** 「全反轉」
- **2補數：** 「反轉加一」
- **整數轉二進制：** 「除二取餘，倒著寫」
- **小數轉二進制：** 「乘二取整，正著寫」
- **格雷轉二進制：** 「高位不動，往下XOR」
- **二進轉格雷：** 「每位XOR前一位」

<a id="常見錯誤提醒"></a>
## 📌 常見錯誤提醒

1. ❌ **忘記2補數要加1** → 只做了反轉（1補數）
2. ❌ **進制轉換餘數順序錯誤** → 整數要由下往上讀
3. ❌ **小數點沒對齊** → 運算前務必補0對齊
4. ❌ **混淆1補數和2補數的進位處理** → 2補數丟棄，1補數循環
5. ❌ **BCD轉換前忘記先轉十進制** → 直接轉會出錯
6. ❌ **格雷碼忘記最高位不變** → 所有位都XOR了
7. ❌ **ASCII混淆大小寫** → 'A'≠'a'，相差32
